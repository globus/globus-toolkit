Myproxy Programs

Preliminaries

The full set of Myproxy programs require either of the following packages be installed:
  • GSI v0.4.0 or later
  • Globus v1.1 or later

Programs

  • myproxy-init
  • myproxy-destroy
  • myproxy-get-delegation
  • myproxy-server

myproxy-init

Creates and sends a delegated proxy to myproxy-server for later use.

SYNOPSIS

myproxy-init [{-h, -u}| {--help, --usage}] [-l | --username username] [-c | --cred_lifetime hours] [-t | --portal_lifetime hours] [-pshost proxy server hostname] [-psport proxy server port #]

DESCRIPTION

A portal user will first enter the passphrase necessary to create a proxy certificate. The user is then prompted for a passphrase to protect the delegated proxy on the myproxy-server. The cred_lifetime is the maximum lifetime of the delegated proxy existing on the myproxy-server. The default value is two weeks. The portal_lifetime is the maximum lifetime a proxy can be delegated to the web server. The default value is 2 hours.

OPTIONS

-h | --help
-u | --usage
Displays usage.
-l | --username
Specifies the username for the delegated proxy.
-c | --cred_lifetime
Can specify the maximum lifetime of the delegated proxy on the myproxy-server in hours
-t | --portal_lifetime
Can specify the maximum lifetime of the delegated proxy to the web server in hours
-s | --pshost
Specifies the hostname of myproxy-server.
-p | --psport
Specifies the port number of myproxy-server.

EXIT STATUS

0 if the delegated proxy was successfully sent to the myproxy-server.
>0 otherwise (If, for instance, there is a naming conflict between the selected username and existing usernames currently known to the MyProxyServer.

myproxy-destroy

Destroy any existing proxy on the myproxy-server.

SYNOPSIS

myproxy-destroy [{-h, -u} | {--help, --usage}] [-l | --username username] [-s | --pshost proxy server hostname] [-p | --psport proxy server port #]

DESCRIPTION

myproxy-destroy removes any delegated proxy certificate from the myproxy-server generated using myproxy-init. The user must enter the pass-phrase that was used during the myproxy-init.

OPTIONS

-h | --help
-u | --usage
Displays usage.
-l | --username
Specifies the username for the delegated proxy.
-s | --pshost
Specifies the hostname of the myproxy-server.
-p | --psport
Specifies the port number of the myproxy-server.

EXIT STATUS

0 if the delegated proxy was deleted from the myproxy-server
>0 otherwise

myproxy-get-delegation

Retrieves a delegated credential from the myproxy-server for use by the user and/or web portal.

SYNOPSIS

myproxy-get-delegation [{-h, -u} | {--help, --usage}] [-l | --username username] [-v | --version] [-t | --portal_lifetime hours] [-o | --out path] [-s | --pshost proxy server hostname] [-p | --psport proxy server port #]

DESCRIPTION

myproxy-get-delegation can be invoked from either a web server or from the user's desktop to retrieve a delegated credential for the given username with the selected pass-phrase. If a valid proxy exists, a delegated proxy will be created for the maximum time dictated by myproxy-init. By default, the delegated proxy is stored in /tmp/ under a unique varying name. The new proxy will be securely delegated using the GSI.

OPTIONS

-h | --help
-u | --usage
Displays usage.
-l | --username
Specifies the username for the delegated proxy.
-portal_lifetime
Can specify the lifetime of the delegated proxy in hours
-s | --pshost
Specifies the hostname of myproxy-server.
-p | --psport
Specifies the port number of myproxy-server.

EXIT STATUS

0 if the proxy was successfully retrieved from the MyProxyServer
>0 otherwise (If, for instance, there is no available proxy)

myproxy-server

Retrieves a delegated credential from myproxy-server for use by the user and/or web portal.

SYNOPSIS

myproxy-server [{-h, -u} | {--help, --usage}] [-p | --port proxy server port #] [-d | --debug] [-c | --config location of myproxy-server configuration file]

DESCRIPTION

myproxy-server is a daemon that runs on a trusted, secure host that manages a database of proxy credentials for use from remote sites, usually web servers. Proxies have a lifetime that is controlled by the myproxy-init program. When a proxy is requested from the myproxy-server, via the myproxy-get-delegation command, further delegation insures that the lifetime of the new proxy may be less than the original to enforce greater security when using the credential from a web server. The myproxy-server uses the GSSAPI library for delegation and authentication between the myproxy-init and myproxy-get-delegation client programs.

A configuration file is responsible for maintaing a list of trusted portals and users that can access this service.

OPTIONS

-h | --help
-u | --usage
Displays usage.
-p | --port
Specifies the port number to service requests
-c | --config
Specifies the location of the configuration file
